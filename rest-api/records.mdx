Use the Records API to manage individual records and bulk record operations in a Kintone app. All endpoints require authentication — see [Authentication](/rest-api/authentication).

| Update Record | PUT | `/k/v1/record.json` |

| Delete Records | DELETE | `/k/v1/records.json` |

## Single record

<AccordionGroup>
  <Accordion title="GET /k/v1/record.json — Get Record">
    Retrieves a single record from an app.

    **Request parameters**

    | Parameter | Type | Required | Description |
    | :-- | :-- | :-- | :-- |
    | `app` | Integer | Yes | The app ID. |
    | `id` | Integer | Yes | The record ID. |

    **Response parameters**

    | Parameter | Type | Description |
    | :-- | :-- | :-- |
    | `record` | Object | An object containing field codes as keys and field value objects as values. |

    <CodeGroup>

    ```bash curl
    curl -X GET 'https://{subdomain}.kintone.com/k/v1/record.json?app=1&id=100' \
      -H 'X-Cybozu-API-Token: {APIToken}' \
      -H 'Content-Type: application/json'
    ```

    ```js kintone.api()
    kintone.api(kintone.api.url('/k/v1/record.json', true), 'GET', {
      app: 1,
      id: 100
    }, function(resp) {
      console.log(resp.record);
    }, function(error) {
      console.log(error);
    });
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {
      "record": {
        "$id": { "type": "__ID__", "value": "100" },
        "$revision": { "type": "__REVISION__", "value": "1" },
        "title": { "type": "SINGLE_LINE_TEXT", "value": "Sample record" }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Multiple records

<AccordionGroup>
  <Accordion title="GET /k/v1/records.json — Get Records">
    Retrieves multiple records from an app using a query string.

    <Note>
      A maximum of 500 records can be retrieved per request. To retrieve more than 500 records, use [cursor-based pagination](#cursor-based-pagination).
    </Note>

    **Request parameters**

    | Parameter | Type | Required | Description |
    | :-- | :-- | :-- | :-- |
    | `app` | Integer | Yes | The app ID. |
    | `query` | String | No | A query string to filter and sort records. |
    | `fields[]` | Array | No | An array of field codes to include in the response. Returns all fields if omitted. |
    | `totalCount` | Boolean | No | If `true`, the total count of records matching the query is returned. Defaults to `false`. |

    **Response parameters**

    | Parameter | Type | Description |
    | :-- | :-- | :-- |
    | `records` | Array | An array of record objects. |
    | `totalCount` | String or null | The total number of records matching the query. `null` if `totalCount` was not set to `true` in the request. |

    <CodeGroup>

    ```bash curl
    curl -X GET 'https://{subdomain}.kintone.com/k/v1/records.json?app=1&query=status%3D%22Open%22%20order%20by%20%24id%20asc%20limit%20100&fields[]=title&fields[]=status&totalCount=true' \
      -H 'X-Cybozu-API-Token: {APIToken}' \
      -H 'Content-Type: application/json'
    ```

    ```js kintone.api()
    kintone.api(kintone.api.url('/k/v1/records.json', true), 'GET', {
      app: 1,
      query: 'status = "Open" order by $id asc limit 100',
      fields: ['title', 'status'],
      totalCount: true
    }, function(resp) {
      console.log(resp.records);
      console.log(resp.totalCount);
    }, function(error) {
      console.log(error);
    });
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {
      "records": [
        {
          "$id": { "type": "__ID__", "value": "1" },
          "title": { "type": "SINGLE_LINE_TEXT", "value": "First record" },
          "status": { "type": "STATUS", "value": "Open" }
        }
      ],
      "totalCount": "42"
    }
    ```
  </Accordion>
</AccordionGroup>

## Update records

<AccordionGroup>
  <Accordion title="PUT /k/v1/record.json — Update Record">

    | Parameter | Type | Required | Description |

    | `revision` | Integer | No | The expected revision number. The update fails if the current revision differs. Omit to skip the revision check. |

    ```bash curl
      -H 'Content-Type: application/json' \
      -d '{
        "app": 1,
        "id": 100,
        "revision": 2,
        "record": {
          "status": { "value": "Closed" }
        }
      }'
    ```

    ```js kintone.api()
    kintone.api(kintone.api.url('/k/v1/record.json', true), 'PUT', {
      app: 1,
      id: 100,
      revision: 2,
      record: {
        status: { value: 'Closed' }
      }
    }, function(resp) {
      console.log(resp);
    }, function(error) {
      console.log(error);
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {
      "revision": "3"
    ```

  </Accordion>

## Cursor-based pagination

When you need to retrieve more than 500 records, use cursors. A cursor stores your query server-side and lets you page through all results.

<Note>
  Cursors expire after 10 minutes of inactivity. Delete a cursor when you are done to free server resources.
</Note>

**Typical flow:**

<Steps>
  <Step title="Create a cursor">
    Call `POST /k/v1/records/cursor.json` with your query. Save the returned `id` and `totalCount`.
  </Step>
  <Step title="Fetch pages">
    Repeatedly call `GET /k/v1/records/cursor.json` with the cursor `id` until `next` is `false`.
  </Step>
  <Step title="Delete the cursor (optional)">
    If you stop early, call `DELETE /k/v1/records/cursor.json` to clean up.
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="POST /k/v1/records/cursor.json — Add Cursor">
    Creates a cursor for paginating through a large record set.

    **Request parameters**

    | Parameter | Type | Required | Description |
    | :-- | :-- | :-- | :-- |
    | `app` | Integer | Yes | The app ID. |
    | `fields[]` | Array | No | An array of field codes to return. Returns all fields if omitted. |
    | `query` | String | No | A query string to filter and sort records. |
    | `size` | Integer | No | Number of records per page. Maximum 500. Defaults to 100. |

    **Response parameters**

    | Parameter | Type | Description |
    | :-- | :-- | :-- |
    | `id` | String | The cursor ID. Use this when calling `GET /k/v1/records/cursor.json`. |
    | `totalCount` | String | The total number of records matching the query. |

    <CodeGroup>

    ```bash curl
    curl -X POST 'https://{subdomain}.kintone.com/k/v1/records/cursor.json' \
      -H 'X-Cybozu-API-Token: {APIToken}' \
      -H 'Content-Type: application/json' \
      -d '{
        "app": 1,
        "query": "status = \"Open\" order by $id asc",
        "fields": ["title", "status"],
        "size": 500
      }'
    ```

    ```js kintone.api()
    kintone.api(kintone.api.url('/k/v1/records/cursor.json', true), 'POST', {
      app: 1,
      query: 'status = "Open" order by $id asc',
      fields: ['title', 'status'],
      size: 500
    }, function(resp) {
      console.log(resp.id);
      console.log(resp.totalCount);
    }, function(error) {
      console.log(error);
    });
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {
      "id": "a8a2d83e-1234-5678-abcd-ef0123456789",
      "totalCount": "1500"
    }
    ```
  </Accordion>

  <Accordion title="GET /k/v1/records/cursor.json — Get Records via Cursor">
    Fetches the next page of records for a cursor. Call repeatedly until `next` is `false`.

    **Request parameters**

    | Parameter | Type | Required | Description |
    | :-- | :-- | :-- | :-- |
    | `id` | String | Yes | The cursor ID returned by Add Cursor. |

    **Response parameters**

    | Parameter | Type | Description |
    | :-- | :-- | :-- |
    | `records` | Array | An array of record objects for this page. |
    | `next` | Boolean | `true` if more pages remain. `false` when this is the last page. The cursor is automatically deleted after the last page is retrieved. |

    <CodeGroup>

    ```bash curl
    curl -X GET 'https://{subdomain}.kintone.com/k/v1/records/cursor.json?id=a8a2d83e-1234-5678-abcd-ef0123456789' \
      -H 'X-Cybozu-API-Token: {APIToken}' \
      -H 'Content-Type: application/json'
    ```

    ```js kintone.api()
    // Fetch all records using a cursor
    async function getAllRecords(app, query) {
      // Create cursor
      const cursor = await kintone.api(
        kintone.api.url('/k/v1/records/cursor.json', true),
        'POST',
        { app, query, size: 500 }
      );

      let allRecords = [];
      let next = true;

      while (next) {
        const page = await kintone.api(
          kintone.api.url('/k/v1/records/cursor.json', true),
          'GET',
          { id: cursor.id }
        );
        allRecords = allRecords.concat(page.records);
        next = page.next;
      }

      return allRecords;
    }
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {
      "records": [
        {
          "$id": { "type": "__ID__", "value": "1" },
          "title": { "type": "SINGLE_LINE_TEXT", "value": "Record one" }
        }
      ],
      "next": true
    }
    ```
  </Accordion>

  <Accordion title="DELETE /k/v1/records/cursor.json — Delete Cursor">
    Deletes a cursor before all records have been retrieved. Use this when you stop paginating early.

    **Request parameters**

    <CodeGroup>

    ```js kintone.api()
      id: 'a8a2d83e-1234-5678-abcd-ef0123456789'
    }, function(resp) {
      console.log(resp);
    }, function(error) {
      console.log(error);
    });
    ```

    </CodeGroup>

    **Sample response**

    ```json
    {}
    ```
  </Accordion>
</AccordionGroup>

## Process management

<AccordionGroup>
  <Accordion title="PUT /k/v1/record/status.json — Update Status">
    Updates the process management status of a single record.

    **Request parameters**

    | Parameter | Type | Required | Description |
    | :-- | :-- | :-- | :-- |
    | `app` | Integer | Yes | The app ID. |
    | `id` | Integer | Yes | The record ID. |
    | `action` | String | Yes | The name of the action to perform (must match an action defined in the app's process management settings). |
    | `assignee` | String | No | The login name of the user to assign. Required if the next status requires an assignee. |
    | `revision` | Integer | No | The expected revision number. The update fails if the current revision differs. |
